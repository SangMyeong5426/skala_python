# 텍스트 기반 미니 RPG

파이썬 입문자를 위한 실습용 텍스트 RPG입니다. 몬스터를 사냥해 레벨업하고, 장비를 모아 더 강한 몬스터에 도전하는 구조입니다.

## 실행 방법

```bash
python3 text_rpg.py
```

Python 3.9 이상이면 동작합니다. 외부 라이브러리 없이 표준 라이브러리(`json`, `random`, `dataclasses`, `pathlib`)만 사용합니다.

## 게임 방법

시작하면 새 게임 또는 저장된 게임 불러오기를 선택합니다. 이후 메인 메뉴에서 아래 행동을 선택할 수 있습니다.

| 번호 | 행동 | 설명 |
|---|---|---|
| 1 | 탐험하기 | 몬스터와 조우해 전투를 시작합니다 |
| 2 | 상점 | 골드로 포션을 구매합니다 (10골드/개) |
| 3 | 상태 보기 | 레벨·HP·공격력·장착 장비를 확인합니다 |
| 4 | 장비 관리 | 미착용 장비를 확인하고 장착 교체합니다 |
| 5 | 저장하기 | 현재 진행 상황을 `savegame.json`에 저장합니다 |
| 6 | 종료 | 게임을 종료합니다 |

전투 중에는 공격 / 포션 사용 / 도망 중 하나를 선택합니다. HP가 0 이하가 되면 게임 오버입니다.

## 핵심 규칙

### 몬스터 출현
몬스터 출현 확률은 **플레이어 레벨과 무관하게 항상 고정**입니다 (`weight` 값의 합이 100).

| 몬스터 | 출현 확률 | 전투력 배율 |
|---|---|---|
| 슬라임 | 10% | 1.0 |
| 고블린 | 15% | 1.4 |
| 놀 | 20% | 1.9 |
| 오크 | 20% | 2.6 |
| 트롤 | 10% | 3.6 |
| 오거 | 10% | 4.8 |
| 마족 | 10% | 6.4 |
| 드래곤 | 5% | 9.0 |

대신 몬스터의 실제 HP·공격력은 **조우하는 순간 플레이어 레벨을 기준으로 다시 계산**됩니다. 전투력 배율이 큰 몬스터일수록 레벨에 따른 스탯 증가폭이 훨씬 커서, 레벨업만으로는 상위 티어 몬스터(트롤 이상)를 이길 수 없습니다. 장비로 보완해야 합니다.

### 장비
몬스터를 처치하면 장비가 하나 드랍됩니다.

- **부위** (6종): 무기, 투구, 상의, 하의, 장갑, 신발
  - 무기: 공격력만
  - 상의·하의: HP만
  - 투구·장갑·신발: 공격력 + HP
- **등급** (드랍 확률 고정): 일반 70%, 고급 16%, 희귀 8%, 영웅 5%, 전설 1%
- 장비 수치는 **획득 당시 레벨 + 등급**에 비례한 범위 내에서 랜덤으로 결정됩니다. 같은 등급이라도 레벨이 높을 때 얻은 장비가 더 강합니다.

드랍 즉시 바로 장착할지 선택할 수 있고, 나중에 "4) 장비 관리"에서 인벤토리에 쌓인 장비로 교체할 수도 있습니다.

## 밸런스 조절하기

`text_rpg.py` 상단의 상수 값만 바꾸면 난이도를 조절할 수 있습니다.

- `MONSTERS`의 `power`: 몬스터별 전투력 배율
- `ATTACK_GROWTH_PER_LEVEL`, `HP_GROWTH_PER_LEVEL`: 레벨당 장비 스탯 성장폭
- `GRADE_MULTIPLIER` (`GRADES` 리스트): 등급별 장비 스탯 배율
- `level_up()` 내부의 `+20`(최대HP), `+5`(공격력): 레벨업 시 캐릭터 기본 스탯 성장폭

## 알려진 제약

- 저장 파일 하나만 지원합니다 (슬롯 여러 개 저장 불가).
- 미착용 장비를 판매하거나 버리는 기능은 없어서, 오래 플레이하면 인벤토리 목록이 계속 늘어납니다.

## 앞으로 추가할 기능 (로드맵)

- **게임 데이터를 JSON으로 분리**: 현재 `MONSTERS`, `EQUIPMENT_PARTS`, `GRADES`는 `text_rpg.py` 안에 파이썬 코드로 하드코딩돼 있습니다. 이를 `data/monsters.json`, `data/equipment.json` 같은 별도 파일로 분리할 예정입니다.
  - 몬스터 스탯이나 등급 확률을 조정할 때마다 코드를 열어 수정·재실행할 필요 없이 데이터 파일만 고치면 됩니다.
  - 로직(`text_rpg.py`)과 데이터(JSON)가 분리되면 둘 중 하나만 바뀌어도 나머지 코드에 영향이 없어 유지보수가 쉬워집니다.
  - 나중에 Pydantic으로 JSON 데이터를 로드 시점에 검증하면, 오타나 잘못된 값이 들어간 데이터 파일을 실행 전에 걸러낼 수 있습니다.
- 저장 슬롯 여러 개 지원 (파일명에 슬롯 번호 부여)
- 미착용 장비 판매·버리기 기능 추가
- 몬스터·장비 데이터 검증 및 게임 로직에 대한 pytest 테스트 추가
