자유 주제 추가과제로, 실제 내 넷플릭스 시청 기록(`NetflixViewingHistory.csv`, 2,205건,


## 왜 이 주제인가 (선택 과정)

원래는 Riot Games API로 LoL 전적을 분석하는 프로젝트를 설계·구현까지 했으나, "숙련도"라는
개념 자체가 API 지표만으로 환원되지 않는 추상적 개념이라는 문제에 부딪혀 방향을 접었다.
대신 이미 확보된 개인 데이터(넷플릭스 시청 기록)로 주제를 바꿨다 - 외부 API 인증 없이도
Day2의 핵심(정제되지 않은 실데이터를 다루는 경험)에 더 집중할 수 있기 때문이다.

## 핵심 설계: 시리즈 vs 영화 분류

Netflix 원본 CSV에는 콘텐츠 타입 구분이 없다. 제목만으로 완벽히 구분하는 규칙은 없지만
(예능 제목처럼 애매한 경우가 많음), 두 단계 휴리스틱으로 근사했다:

1. 같은 `show_key`(제목의 첫 `:` 앞부분)가 2번 이상 나오면 → "시리즈" (반복 시청 자체가
   강한 신호)
2. 1번에 안 걸려도 제목에 회차 마커(`N화`, `시즌 N`, `파트 N`, `YYYY-MM-DD` 등)가 있으면
   → "시리즈"
3. 둘 다 아니면 → "영화/단편"

빈 제목(`" : 24화"` 같은 Netflix 메타데이터 누락 레코드, 36건)은 "제목정보없음"으로
별도 분류해 통계를 왜곡하지 않도록 했다.

## pandas ↔ Polars 교차검증

ex05(Polars/DuckDB)에서 배운 "같은 집계를 다른 엔진으로 각각 구해 결과가 일치하는지
검증한다"는 원칙을 그대로 적용했다. `verify_with_polars()`가 원본 CSV를 Polars로 직접
읽어 show_key를 뽑고 집계한 뒤, pandas가 낸 Top N과 `assert`로 비교한다. 2,205행짜리
소규모 데이터라 두 엔진의 속도 차이는 ex05(대용량)만큼 크게 벌어지지 않지만
(pandas 0.13ms vs Polars 0.31ms, 5회 중 최소값 - 이 규모에서는 오히려 pandas가
더 빠르다), 이 비교의 핵심은 속도가 아니라 "같은 결론에 도달하는가"를 코드로
검증했다는 데 있다.

## 실행 결과 (2026-07-21 기준)

- 총 시청 레코드: 2,205건 (시리즈 2,020 · 영화/단편 149 · 제목정보없음 36)
- Top 콘텐츠: 나의 히어로 아카데미아(108화) · 전생했더니 슬라임이었던 건에 대하여(50화) ·
  블랙 클로버(48화) · 불꽃 소방대(48화) · 주술회전(45화)
- 몰아본 날(하루 3화 이상): 236일. 최고 기록은 2024-11-20, "던전에서 만남을 추구하면
  안 되는 걸까" 29화 연속 시청
- t-검정 (주말 vs 평일 일일 시청량): p=2.16e-01 → **유의하지 않음** (주말 평균 3.57화 /
  평일 평균 4.00화)
- 카이제곱 적합도 (요일별 균등분포 여부): p=7.22e-06 → **유의함** (금요일에 가장 몰림,
  목요일에 가장 적음)

## 잘된 점

- 두 통계 검정이 서로 다른 질문을 던진다는 점이 실제로 드러났다: "주말/평일 이분법"으로는
  차이가 안 보이지만(t-검정), "요일별 세분화"로는 유의한 편차가 있다(카이제곱). 검정 방법에
  따라 같은 데이터에서 다른 결론이 나올 수 있다는 걸 보여주는 좋은 사례가 됐다.
- 빈도 기반 분류 규칙이 예능/애니메이션처럼 제목 패턴이 제각각인 콘텐츠에도 잘 작동했다
  (오분류율을 정량적으로 검증하진 않았지만, Top 10 콘텐츠가 전부 실제 시리즈로 정확히
  분류됨을 육안으로 확인).
- pandas로 낸 결과를 "믿고 넘어가는" 대신 Polars로 독립적으로 재계산해 `assert`로
  검증했다 - 실습에서 배운 검증 습관을 실제로 적용한 부분.
